Terrain Generation

By the Preposterous Particles:

Jaron Erba, Jennifer Zhao, Sriram Srivatsan, and Lili Wang

Abstract

Many popular games utilize aspects of terrain generation to create a realistic, yet non-deterministic, experience. The most prominent example of such a game is Minecraft, where a procedural generation script is ran upon creation of a game instance, dynamically updating the terrain as more of the world is explored. Similarly, other games like No Man's Sky use algorithms like Marching Cubes and the Transvoxel Algorithm in order to make the world look more realistic instead of block-y while also minimizing computational and storage overhead. In this project, we implemented many popular terrain generation algorithms such as 2D and 3D Perlin Noise and Marching Cubes, incorporating our own ideas in order to adapt the generation for the purpose of making unique and interesting scenes.

Technical Approach

Unity, Blender, and Sources

Due to the overhead of creating our own rendering application or building from a CS 184 project implementation, we decided to use Unity in order to help with rendering our objects and having datastructures we could modify. To ensure that we did a substantial amount of work, we limited ourselves to only using Unity to render, and we coded all GameObjects, algorithms, assets, GPU accelerations, and shaders by ourselves. Unfortunately, learning Unity was probably half of the work in this project, since there was a lot of documentation to read and videos to watch before we could even start with the coding. We also used Blender to model trees and leaves, which we used as surface objects in order to make our terrain more realistic. We made these assets ourselves, without downloading or using any already existing models. Finally, we referenced many articles and videos throughout our implementation. We did not borrow any code except for a list of all possible marching cubes configurations which is a static table, but we used many ideas which are cited at the bottom of this page.

2D Perlin Noise

Our first goal was to make a realistic procedurally generated surface. To make this surface, first we made a mesh of vertices, each with an $x$ and $z$ position where $-\frac{s}{2} \leq x, z \leq \frac{s}{2}$, where $s$ is the size of the grid. There are $n x n$ total vertices where $\frac{n}{s}$ is the resolution of the mesh. To create mountains and valleys and other dynamic objects, we then sampled the $Mathf.PerlinNoise$ function on each vertex's coordinate to get a $y$-value associated with it. 3D Perlin Noise

Marching cubes

The base of the terrain is generated using the Marching Cubes Algorithm. The Marching Cubes Algorithm fills a 3D grid in space with a polygon mesh surface. The 3D grid is iterated through cube by cube, with each cube adjacent to 8 vertices. (Also known as voxels.) Each vertex is determined to be within or above the surface of the terrain based on a threshold value. The Marching Cubes Algorithm then uses a lookup table to determine which edges of the cube are intersected by the surface and creates triangles based on the intersection points.

Particle Simulation

For the particle system, we created a particle class with attributes such as position, velocity, and radius. The particles are initialized with a random position within the plane and added to a list of particles. Each particle starts with an initial downward velocity due to gravity and the particle position is updated based on the current velocity. We used the terrain vertices to determine when the particle collided with the terrain and bounced the particle off the ground with a certain damping factor.

We checked the distance between each particle and moved the particles randomly to the side if there was a collision to simulate elastic collisions between the particles. We attempted to use terrain tangent vectors to calculate changes in direction due to collisions but we were not able to complete this in time.

We added the particles to Unity by creating a list of prefabs which are located at the positions defined by the particles in the particles list. We modeled petals and hail in Blender and imported them into Unity, using the models as the object for the prefab.
